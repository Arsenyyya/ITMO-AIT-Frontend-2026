# Домашняя работа 2 (вариант 4)

Тема: улучшение HTML-доступности сайта сервиса личных финансов.

## Что сделано

- Взята рабочая версия сайта и улучшена доступность интерфейса.
- На все страницы добавлена `skip-link` («Перейти к основному содержимому»).
- Главный контейнер контента размечен как фокусируемая область (`id="mainContent"`, `tabindex="-1"`).
- Добавлены и усилены стили `:focus-visible` для ссылок, кнопок и полей формы.
- Для динамически обновляемых блоков добавлены `aria-live` и `aria-atomic`.
- Улучшены роли и озвучивание уведомлений: `status`/`alert` для сообщений форм и глобальных ошибок.
- Добавлена поддержка `prefers-reduced-motion` для пользователей с ограничением анимаций.

## Структура

- `index.html`, `login.html`, `register.html`, `dashboard.html`, `transactions.html`, `report.html`, `integrations.html`.
- `assets/css/styles.css` — визуальные стили + accessibility-улучшения.
- `assets/js/script.js` — логика приложения + ARIA/live-region настройки.
- `db.json`, `routes.json`, `package.json` — mock API для демонстрации интерактивных страниц.

## Проверка

1. На любой странице нажать `Tab` — должна появиться `skip-link`.
2. Активировать `skip-link` — фокус должен перейти в основное содержимое.
3. Проверить видимую фокус-рамку на ссылках, кнопках и инпутах.
4. В формах входа/регистрации вызвать ошибку — сообщение должно отображаться как `alert`.
5. На страницах с динамическими данными (`transactions`, `report`, `integrations`) проверить корректные обновления без потери доступности.
