# Лабораторная работа 3 (вариант 4)

Тема: сервис для управления личными финансами и бюджетом (миграция на Vue SPA).

## Что реализовано

- Приложение из ЛР2 мигрировано на Vue 3 + Vite.
- Подключен `vue-router` с `meta` и `navigation guards`:
  - приватные страницы доступны только авторизованному пользователю;
  - страницы входа/регистрации недоступны после авторизации.
- Работа с внешним API выполнена через `axios`.
- Логика авторизации и сессии вынесена в composable `useAuth`.
- Повторно используемые финансовые утилиты вынесены в composable `useFinanceHelpers`.
- Реализованы страницы:
  - `home`
  - `login`
  - `register`
  - `dashboard`
  - `transactions`
  - `report`
  - `integrations`
- Подключен mock API на `json-server` + `json-server-auth` (`/login`, `/register`, JWT, приватные коллекции).

## Структура

- `src/router/index.js` — маршруты + guard + title.
- `src/composables/useAuth.js` — сессия, вход, регистрация, гидратация пользователя.
- `src/composables/useFinanceHelpers.js` — форматирование и расчетные утилиты.
- `src/services/api.js` — axios-клиент и API-функции.
- `src/views/*` — страницы SPA.
- `src/components/*` — переиспользуемые компоненты.
- `db.json`, `routes.json` — mock API и правила доступа.

## Запуск

1. Перейти в директорию:
   ```bash
   cd "J3210/Павлов Арсений/Labs/Lab3"
   ```
2. Установить зависимости:
   ```bash
   npm install
   ```
3. Запустить mock API:
   ```bash
   npm run api
   ```
4. В отдельном терминале запустить SPA:
   ```bash
   npm run dev
   ```
5. Открыть ссылку из Vite (обычно `http://localhost:5173`).

## Быстрая проверка

1. Открыть `/register`, создать нового пользователя.
2. Убедиться, что произошел редирект в `/dashboard`.
3. Добавить расход в кабинете и проверить изменение списка транзакций.
4. На `/transactions` проверить фильтрацию.
5. На `/report` проверить расчет показателей и график по категориям.
6. На `/integrations` подключить/обновить провайдера и проверить журнал синхронизации.
