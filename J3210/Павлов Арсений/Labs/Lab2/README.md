# Лабораторная работа 2 (вариант 4)

Тема: сервис для управления личными финансами и бюджетом.

## Что сделано

- Приложение из ЛР1 привязано к внешнему API через `fetch`.
- Реализовано моковое API на `json-server`.
- Подключена авторизация через `json-server-auth` (`/register`, `/login`, JWT).
- Данные пользователя (счета, транзакции, бюджеты, интеграции) читаются и изменяются через API.
- Реализована защита приватных страниц (кабинет, транзакции, отчеты, интеграции).
- `db.json` может быть пустым или содержать ранее созданные данные; при первом входе нового пользователя стартовые данные создаются автоматически.

## Структура

- `index.html`, `login.html`, `register.html`, `dashboard.html`, `transactions.html`, `report.html`, `integrations.html` — страницы интерфейса.
- `assets/css/styles.css` — оформление и адаптивность.
- `assets/js/script.js` — логика API, авторизация, фильтрация, рендер данных.
- `db.json` — mock база данных.
- `routes.json` — ограничения доступа к ресурсам по токену.
- `package.json` — запуск JSON-сервера.

## Запуск

1. Перейти в директорию ЛР2:
   ```bash
   cd "J3210/Павлов Арсений/Labs/Lab2"
   ```
2. Установить зависимости:
   ```bash
   npm install
   ```
3. Запустить mock API:
   ```bash
   npm run api
   ```
4. Открыть `index.html` (желательно через Live Server / локальный http-server).

API по умолчанию работает на `http://localhost:3000`.

## Мини-сценарий проверки

1. Открыть страницу регистрации и создать пользователя.
2. После входа проверить загрузку кабинета (счета, бюджеты, транзакции).
3. Добавить расход в модальном окне на странице кабинета.
4. Перейти на страницу поиска и проверить фильтрацию.
5. Открыть отчеты и убедиться, что график и суммы строятся по API-данным.
6. На странице интеграций подключить/обновить провайдера через модальное окно.
